# Исправление проблемы сборки Docker

## Текущая проблема
Docker не может собрать образ из-за сетевых проблем при установке npm пакетов.

## Решения (попробуйте по порядку):

### 1. Проверьте интернет-соединение
Убедитесь, что у вас стабильное интернет-соединение.

### 2. Отключите прокси в Docker Desktop
1. Откройте Docker Desktop
2. Settings → Resources → Proxies
3. Отключите прокси (если не нужен)
4. Нажмите "Apply & Restart"

### 3. Попробуйте собрать в другое время
Иногда npm registry может быть перегружен.

### 4. Используйте другой DNS
В Docker Desktop:
1. Settings → Docker Engine
2. Добавьте:
```json
{
  "dns": ["8.8.8.8", "8.8.4.4"]
}
```
3. Нажмите "Apply & Restart"

### 5. Временно отключите файрвол
Проверьте, не блокирует ли файрвол Docker.

### 6. Попробуйте собрать только базу данных
База данных уже работает:
```powershell
docker-compose up -d postgres
```

## Текущий статус
- ✅ База данных (postgres) работает
- ❌ Backend не собирается из-за npm install таймаута
- ❌ Frontend не собирается из-за npm install таймаута

## Альтернативное решение
Если проблема с сетью сохраняется, можно:
1. Собрать образ на другой машине с хорошим интернетом
2. Или использовать готовый образ (если есть)
3. Или запустить приложение без Docker (локально)

## После исправления сети
Выполните:
```powershell
docker-compose build --no-cache
docker-compose up -d
```
